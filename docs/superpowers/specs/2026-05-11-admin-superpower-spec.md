# Admin Superpower Spec — FanAPI

**Date:** 2026-05-11
**Scope:** Admin 控制台功能增强与新模块设计规格
**Priority:** 按模块逐步实现，优先已有页面的能力补全，再补新模块

---

## 总体原则

- 所有新功能必须遵守 `DESIGN.md` 约束：语义 Token、shadcn/ui 组件、共享布局 shell
- 写操作必须留痕（操作人 + 时间 + 前后值），涉及资金的操作需二次确认
- 导出类操作若数据量大，采用异步任务 + 下载链接模式（见"数据导出中心"）
- 权限管控基于 RBAC 模型（见"权限角色"模块），菜单动态渲染

## 一、数据概览（Dashboard）

### 1.1 趋势曲线

| 字段 | 说明 |
|------|------|
| 时间范围 | 近 7 天 / 近 30 天，可切换 |
| 维度切换 | 收入 / 成本 / 利润 / 调用量，单维度折线图 |
| 数据粒度 | 按天 |
| 刷新 | 页面加载时请求，支持手动刷新 |

交互：维度切换 Tab + 时间范围 SegmentedControl，图表使用统一图表组件（建议 Recharts）。

### 1.2 TOP 榜

| 榜单 | 维度 | 数量 |
|------|------|------|
| 消耗用户 | 今日消费金额 TOP | 10 |
| 模型 | 今日调用量 TOP | 10 |
| 渠道 | 今日调用量 TOP | 10 |

展示形式：简洁 ranked list，支持点击跳转到对应详情页。

## 二、渠道管理（Channels）

### 2.1 渠道分组

- 支持创建分组标签（高质 / 低价 / 备用 / 自定义）
- 渠道可绑定多个分组标签
- 路由策略可基于分组（结合用户定价分组）进行配置

### 2.2 健康监控

| 指标 | 说明 |
|------|------|
| 24h 成功率 | 按渠道展示成功率进度条 |
| 平均延迟 | 近 24h P50/P99 延迟 ms |
| 失败原因 TOP | 错误码 / 错误类型 TOP5，可下钻 |

展示位置：渠道列表行内展开 或 详情侧边栏。

### 2.3 批量操作

- 批量启停：多选 → 批量启用/禁用，弹出确认对话框
- 批量调倍率：多选 → 弹出倍率输入表单，批量设置

### 2.4 变更日志

每次渠道倍率、状态、分组变更自动记录：

| 字段 | 说明 |
|------|------|
| 操作人 | 管理员 ID + 邮箱 |
| 时间 | 精确到秒 |
| 变更字段 | 字段名 |
| 前值 | 变更前的值 |
| 后值 | 变更后的值 |

渠道详情页内嵌变更日志 Tab。

## 三、号池管理（Key Pools）

### 3.1 Key 详情

每个 Key 在列表/详情页展示：

| 字段 | 说明 |
|------|------|
| 余额 | 同步自上游（见上游平台管理） |
| 累计调用 | 平台侧统计 |
| 失败率 | 近 24h |
| 最后使用时间 | 精确到秒 |
| 来源号商 | 关联 vendor 信息 |

### 3.2 批量导入导出

- **批量导入**：支持 CSV 和纯文本（每行一个 Key）格式上传
- **批量导出**：导出明文 Key 需要二次身份校验（当前密码确认）

### 3.3 绑定关系可视化

- 展示 `号池 → 渠道 → 模型` 的引用关系图
- 删除号池或 Key 时，若有渠道/模型正在引用，弹出警告并列出影响面

## 四、用户管理（Users）

### 4.1 复合筛选

支持以下条件组合查询：

| 字段 | 控件类型 |
|------|----------|
| 邮箱 | 文本输入（模糊匹配） |
| UID | 精确输入 |
| 余额范围 | 数值区间 |
| 注册时间 | 日期区间 |
| 分组 | 多选下拉 |
| 状态 | 正常 / 冻结 / 多选 |

### 4.2 批量操作

- 批量改分组：多选用户 → 选择目标分组
- 批量赠分：多选用户 → 输入赠送金额 + 原因（活动备注）
- 批量冻结：多选用户 → 确认对话框 + 原因输入

### 4.3 消费画像（用户详情页扩展）

| 区块 | 内容 |
|------|------|
| 30 天消费曲线 | 按日消费折线图 |
| 常用模型 | TOP5 模型 + 调用量 |
| API Key 列表 | 该用户所有 Key，含活跃状态 |
| 登录 IP/设备 | 最近登录 IP、UA 摘要 |

### 4.4 风控标签

自动标记规则（后端异步任务）：

| 标签 | 触发条件 |
|------|----------|
| 同 IP 多号 | 同一 IP 注册 ≥3 个账号 |
| 薅羊毛嫌疑 | 短时间内多次兑换卡密或领取优惠 |
| 短时高额 | 24h 内消费超过阈值（可配置） |

人工打标：在用户详情页可手动添加/移除风控标签，需填写原因，留操作日志。

### 4.5 操作日志（用户详情页）

用户详情页直接展示该用户的所有手动操作历史：

- 手动充值 / 扣款
- 冻结 / 解冻
- 分组变更
- 人工打标

## 五、账单流水（Billing）

### 5.1 多维聚合视图

支持以下聚合维度（可切换 Tab）：

- 按用户聚合
- 按渠道聚合
- 按模型聚合
- 按日聚合

### 5.2 对账单导出

- 支持按月份选择导出范围
- 导出格式：CSV / Excel
- 大数据量走异步导出（见"数据导出中心"）

### 5.3 类型筛选

支持按账单类型过滤：

- 结算
- 预扣
- 退款
- 充值
- 提现
- 调整

### 5.4 关联跳转

- 账单行中的关联 ID（如 llm_log_id / task_id）可点击跳转到对应详情

### 5.5 手动调账

专用入口（需财务权限）：

| 字段 | 说明 |
|------|------|
| 用户 | 搜索选择 |
| 类型 | 补单 / 冲销 |
| 金额 | 正负均可 |
| 原因 | 强制填写（≥10字） |
| 操作人 | 自动记录当前管理员 |

## 六、任务中心（Tasks）

### 6.1 失败原因列

任务列表直接展示 `error` 字段摘要（截断 100 字符，悬浮展开）。

### 6.2 失败统计卡片

页面顶部统计区：

- 今日成功率
- 今日平均耗时（ms）
- 失败 TOP 原因（TOP3 错误类型 + 数量）

### 6.3 失败告警

可配置阈值：单位时间（如 5min）内失败率超过 X%，触发推送。

推送渠道：

- 邮件
- 企业微信 Webhook（可配置 URL）

## 七、卡密管理（Cards）

### 7.1 批次管理

| 字段 | 说明 |
|------|------|
| 批次号 | 自动生成 UUID |
| 用途备注 | 手动填写（如"五一活动"） |
| 生成人 | 当前管理员 |
| 数量 | 批次内卡密数量 |
| 生成时间 | 精确到秒 |

列表支持按批次分组展示，可展开查看批次内卡密列表。

### 7.2 导出未用卡密

- 可按批次筛选导出
- 导出内容：卡密明文 + 面值 + 批次号
- 需要二次确认（操作人身份校验）

### 7.3 作废操作

- 单张作废：在卡密详情页操作
- 整批作废：在批次详情页批量操作
- 已使用的卡密不可作废，作废操作留日志

### 7.4 绑定分销返佣

- 卡密生成时可绑定来源分销（vendor ID）
- 卡密被兑换后，自动触发返佣计算（返佣比例在 vendor 设置中配置）

### 7.5 使用记录

卡密详情页展示兑换记录：

| 字段 | 说明 |
|------|------|
| 兑换用户 | UID + 邮箱 |
| 兑换时间 | 精确到秒 |
| 兑换 IP | 来源 IP |

## 八、提现管理（Withdraw）

### 8.1 两段审批流程

```
提现申请 → 客服初审（通过/驳回）→ 财务复审（通过/驳回）→ 打款 → 完结
```

每步操作自动记录：操作人 + 时间 + 备注。

### 8.2 打款凭证

财务打款后需上传：

- 转账截图（图片上传）
- 或银行流水号（文本输入）

凭证与提现记录绑定，对账可查。

### 8.3 限额规则（可配置）

| 规则 | 说明 |
|------|------|
| 单笔上限 | 单次提现不超过 X 元 |
| 单日上限 | 单日累计提现不超过 Y 元 |
| 新号商首笔限额 | 注册后 N 天内首笔不超过 Z 元 |

限额配置在系统设置中统一管理。

### 8.4 驳回原因

驳回时从标准化原因清单选择（可多选 + 自定义补充）：

- 收款信息不完整
- 金额超出当日限额
- 账户存在风险标记
- 材料缺失（需补交）
- 其他（必须填写说明）

驳回后自动通知号商（站内信 + 可选邮件）。

## 十、新增模块

### 10.1 全局操作审计（Audit Log）

**覆盖范围：** 所有管理员写操作（POST / PUT / PATCH / DELETE）

| 字段 | 说明 |
|------|------|
| 操作人 | 管理员 ID + 邮箱 |
| 时间 | 精确到毫秒 |
| 操作类型 | 创建 / 修改 / 删除 / 批量操作 |
| 资源类型 | 用户 / 渠道 / 号池 / 卡密 / 账单等 |
| 资源 ID | 受影响的对象 ID |
| 前值 | JSON diff |
| 后值 | JSON diff |
| IP | 来源 IP |
| UA | User-Agent 摘要 |

UI：独立的审计日志页面，支持按操作人/时间/资源类型过滤，只读不可删除。

---

### 10.2 RBAC 权限角色

**模型：** 三层结构

```
Permission（权限点）
     ↓
  Role（角色）— 绑定若干权限点
     ↓
AdminUser（管理员账号）— 绑定若干角色
```

**权限点示例：**

| 权限点 ID | 说明 |
|-----------|------|
| `channel:read` | 查看渠道列表 |
| `channel:write` | 新建/编辑渠道 |
| `user:freeze` | 冻结用户 |
| `billing:adjust` | 手动调账 |
| `withdraw:approve` | 提现审批 |
| `audit:read` | 查看审计日志 |
| `settings:write` | 修改系统设置 |

**内置角色：**

- 超级管理员（全部权限）
- 运营（用户管理、卡密、渠道读）
- 财务（账单、提现审批、手动调账）
- 客服（用户查询、提现初审、站内信）
- 只读（全部读权限，无写权限）

**菜单动态渲染：** 根据当前管理员的权限点，动态显示/隐藏左侧导航菜单项。

---

### 10.3 通知中心（Notifications）

**功能：**

- 站内信：管理员发送给指定用户的消息（单发 / 群发）
- 维护通知：系统维护公告，带有效期，用户端 Banner 展示
- Key 异常实时通知：当 Key 余额不足/失效时，通知关联号商
- 自定义定向推送：按用户分组批量推送

**管理界面：**

- 通知列表（已发 / 草稿 / 定时）
- 创建通知表单（标题、内容富文本、接收范围、定时发送）
- 发送记录及已读状态统计

---

### 10.4 优惠券（Coupons）

与卡密区分：优惠券不是面值直冲，而是消费优惠。

**类型：**

| 类型 | 说明 |
|------|------|
| 满减券 | 消费满 X 元减 Y 元 |
| 折扣券 | 按比例打折，上限可设 |
| 赠送券 | 使用后直接赠送余额（等同于小额卡密的软化形式） |

**管理功能：**

- 创建批次（设置类型、面值、有效期、发放数量上限、每人领取上限）
- 定向发放给指定用户/分组
- 领取记录 + 使用记录
- 作废整批次

---

### 10.5 用户 API Key 总览（Platform Key View）

平台维度（跨所有用户）查看 API Key：

| 字段 | 说明 |
|------|------|
| Key ID | 脱敏展示（前8位 + ***） |
| 所属用户 | UID + 邮箱 |
| 最后使用时间 | 精确到秒 |
| 近 7 天调用量 | — |
| 状态 | 活跃 / 不活跃 / 已吊销 |

支持：

- 按用户 / 活跃状态筛选
- 一键吊销（需确认 + 原因）

---

### 10.6 告警中心（Alert Center）

统一展示所有告警：

| 告警类型 | 触发条件 |
|----------|----------|
| 渠道异常 | 成功率 < 阈值 / 全量失败 |
| 失败率告警 | 平台整体失败率超阈值 |
| 利润异常 | 某模型成本 > 收入 |
| 资金告警 | 号商余额不足 X 元 |

**告警记录：**

- 告警列表（时间 / 类型 / 状态：未处理 / 已确认 / 已解决）
- 告警详情（触发数据快照）
- 确认/解决操作（留处理人 + 备注）

**通知渠道配置（在系统设置中）：**

- 邮件
- 企业微信 Webhook
- 钉钉 Webhook

---

### 10.7 数据导出中心（Async Export）

所有大数据量导出走统一异步任务流：

**流程：**

```
用户发起导出 → 后端创建导出任务 → 异步生成文件 → 文件就绪通知 → 用户下载
```

**导出中心 UI：**

| 字段 | 说明 |
|------|------|
| 任务名称 | 如"账单流水 2026-04 导出" |
| 状态 | 排队中 / 生成中 / 已完成 / 失败 |
| 进度 | 百分比 |
| 文件大小 | 完成后展示 |
| 创建时间 | — |
| 下载链接 | 完成后有效，过期时间 24h |

文件过期后链接失效，需重新发起导出。

---

### 10.8 客户充值明细

财务专用页面：

| 字段 | 说明 |
|------|------|
| 客户 ID | UID + 邮箱 |
| 金额 | 充值金额（元） |
| 时间 | 精确到秒 |
| 支付方式 | 微信 / 支付宝 / 卡密 / 手动 |
| 是否到账 | 状态标记 |
| 备注 | 手动备注 |

支持按支付方式 / 时间范围 / 到账状态筛选，支持导出（走异步导出）。

---

### 10.9 上游平台管理（Upstream Platforms）

管理上游 AI 供应商账号及能力同步。

**账号管理：**

| 字段 | 说明 |
|------|------|
| 平台名称 | OpenAI / Anthropic / 自定义 |
| API Key | 脱敏存储 |
| Base URL | 上游请求地址 |
| 同步状态 | 正常 / 异常 / 从未同步 |
| 最后同步时间 | — |

**功能：**

1. **自动同步余额**：定时（如每小时）拉取上游账号余额，展示在列表
2. **自动获取模型列表**：调用上游 `/models` 接口，拉取可用模型
3. **一键添加渠道**：根据拉取到的模型列表，选择目标模型后一键在渠道管理中批量创建渠道，预填参数（模型名、Base URL、Key）

## 实现优先级建议

### P0（已有页面补全，改动小，价值高）

1. 用户管理：复合筛选、批量操作、消费画像详情页
2. 渠道管理：健康监控、批量操作、变更日志
3. 账单流水：类型筛选、多维聚合、关联跳转
4. 任务中心：失败原因列、失败统计卡片
5. 数据概览：趋势曲线、TOP 榜

### P1（新功能，依赖已有基础设施）

6. 卡密管理：批次管理、使用记录、作废操作
7. 提现管理：两段审批流、打款凭证
8. 系统设置：操作日志
9. 客户充值明细
10. 上游平台管理

### P2（新模块，需后端配合）

11. 数据导出中心
12. 全局操作审计
13. 告警中心
14. 通知中心
15. 用户 API Key 总览

### P3（较大架构变动）

16. RBAC 权限角色
17. 优惠券模块
18. 风控标签（需后端异步检测任务）
19. 号池绑定关系可视化
